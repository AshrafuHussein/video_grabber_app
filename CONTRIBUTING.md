# Contributing to GrabIt

Thanks for your interest in contributing to GrabIt. Contributions of all kinds are welcome, including bug reports, feature ideas, documentation improvements, and code changes.

## How to Contribute

1. Fork the repository.
2. Create a new branch for your change.
3. Make your changes and test them locally.
4. Open a pull request with a clear description of what was changed and why.

## Code Style

- Follow the existing Dart and Flutter style used in the project.
- Keep changes focused and easy to review.
- Write clear commit messages.

## Before Submitting a Pull Request

- Ensure the app still builds and runs locally.
- If you add a feature, describe its purpose and how it works.
- If you fix a bug, include steps to reproduce it or context for the issue.

## Reporting Issues

Please open an issue if you notice:

- a bug or crash
- a missing feature
- unclear documentation
- unexpected behavior

## Community Guidelines

Be respectful, constructive, and open to feedback. We welcome contributions from beginners and experienced developers alike.
